Highlight #1 Git basics

Other storage systems store information as a list of file-based changes. Git stores data as a series of snapshots of a smaller filesystem. In summary, Git basically stores a snapshot of what your files look like at that moment and stores that photo to use as a reference. And it only saves a new screenshot when the files change. Mostly all Git operations are local and store the entirety of a project on the local disk so operations move much more quickly. Basically, Git does not have to leave the local disk or go out into the server to retrieve information. Previously, I did not know anything about Git so it is interesting to learn something new and it is important to understand the fundamentals of what Git is for this class because we will be using it throughout the class.

Highlight #2 A short history of Git

Git was developed through a collaboration of Linux systems, an open source software and a propriety DVCS, distributed version control, called BitKeeper. These two open source software systems were once able to collaborate free of charge but once this relationship dissolved, Linux took it into their own hands to develop a tool similar to BitKeeper that they could use. Git has been extremely helpful for its users thanks to its speed, simple design, and ability to support large projects. Because we will be using Git in this class it is important to understand what is it and where it came from. Why the developers of Git needed a tool like it and in what ways we can also benefit from this tool.

Highlight #3 About version control

To understand Git you must understand the fundamentals of version control first. Version control is a system that stores changes to files and file sets overtime so one can go back and recall older versions at any time. Version control systems (VCS) are helpful because they not only stores all your history such as downloads, images, recent modifications to a document and if a document is lost or deleted, a VCS allows for an easy recovery of that document. Understanding version control is important because it is something that should be used when dealing with sensitive data or working on an important project. During this course we will be working on many projects and will be using many files, zips, and other documents so it will be important to keep track of those.

Highlight #4 Recording changes to the repository

Usually when using Git one will start recording snapshots of their changes each time the project reaches a state that needs to be recorded. Every file in a working directory is either "tracked" or "untracked". "Tracked" files are basically files that Git knows about that can be modified or unmodified. When a repository is first cloned it will be tracked and unmodified because nothing has been changed yet. As the files are changed then Git updates them using version control systems. Because we will be working with repositories in this class and editing them it is important to understand what is going on behind the scenes and how and why they are being saved after every change.

Highlight #5 Learn Git in 7 minutes

There are a couple different functions that Git preforms such as log, status, blame_, and diff. Git log shows all the backups created in the repository. Git status shows the status of each folder and when they were last changed, backed up, and added to the staging area. Git blame_ is similar to what it sounds like, it records who wrote which line of code or who is to be blamed for a certain line of code. Git diff shows the difference between the older saved repositories and the current directory. Understanding the basic fundamentals of Git is important because we will be working with Git a lot in this class and previously I had no knowledge about Git.
